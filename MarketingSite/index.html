The problem

Emergency help often gets slowed down by fragmented information.

During hurricanes, floods, earthquakes, fires, and medical emergencies, people may report needs through hotlines, social media, direct messages, or paper logs. Coordinators then have to guess where the worst needs are, which volunteers are available, and which supplies are already deployed.

ReddiHelp solves that by connecting incident reporting, GIS mapping, resource tracking, volunteer dispatch, alerts, and analytics in one coordinated system.

What it does

One platform for reporting, dispatch, and coordination.

The mobile app supports field users under pressure, while the web dashboard gives coordinators a command center for the whole response.

How it works

From distress report to resolved incident.

  1. 01

    Report

    A victim submits an SOS or incident report with location, category, severity, notes, and optional media.

  2. 02

    Map

    The backend stores the event with PostGIS and broadcasts live updates to mobile clients and the dashboard.

  3. 03

    Dispatch

    Coordinators match nearby volunteers or responders by availability, distance, role, and relevant resources.

  4. 04

    Respond

    Field users accept tasks, navigate to the scene, update progress, and communicate through real-time channels.

  5. 05

    Analyze

    Reports and dashboards help teams understand response trends, resource use, and remaining gaps.

How it is used

Every user sees the tools they need.

ReddiHelp uses role-based access control so each group gets a focused workflow instead of a one-size-fits-all interface.

Victim

Requests urgent help from a simple mobile interface, shares current location, and tracks whether a report has been received or assigned.

  • Fast SOS and incident submission
  • Offline queue for poor connectivity
  • Status updates as help is coordinated

How it is built

A connected mobile, web, and backend system.

The project combines Flutter, React, Node.js, PostgreSQL, PostGIS, Socket.io, Firebase services, map layers, and deployment-ready environment configuration.

Mobile field app

Flutter screens cover incident reporting, maps, preparedness guides, volunteer tasks, responder views, broadcast alerts, and accessibility settings.

Coordinator dashboard

React and Vite power a web command center for incidents, resources, volunteers, broadcasts, reports, analytics, and public status.

Real-time backend

Node.js and Express expose REST APIs, handle authentication, store geospatial data, stream WebSocket events, and manage notifications.

Why use it

ReddiHelp makes scarce emergency resources easier to see, assign, and account for.

Faster response

GPS-tagged reports and live dispatch reduce the time spent locating victims and manually relaying updates.

Better resource use

Teams can see available supplies, open needs, and active deployments before sending help to the wrong place.

Clear accountability

Incident status, assignments, history, and reports create a shared record from first report to resolution.

More equitable coverage

A centralized map helps coordinators spot underserved areas and support vulnerable communities during a crisis.

What was accomplished

Implemented and tested around real response scenarios.

  • Incident reports can be created, assigned, updated, and resolved.
  • WebSocket updates appear on other clients without manual refresh.
  • Volunteers can manage profile skills, availability, assignments, and achievements.
  • Coordinators can generate analytics and export reports for review.

What comes next

Ready for smarter forecasting and larger-scale operations.

Predictive analytics: Forecast incident growth and resource depletion from historical data.

Enhanced maps: Add weather, traffic, demographics, and operational overlays for richer decisions.

Intelligent dispatch: Refine automated matching with confidence scoring and decision support.
